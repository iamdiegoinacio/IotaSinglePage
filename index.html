Tangle is a distributed ledger technology (in English, distributed ledger) with architecture based on directed acyclic graphs (DAG), unlike its originated technology, Blockchains, which is characterized by being a finite-sized block chain, which can only be produced (and validated) at regular discrete time intervals. Therefore, the architecture of the Blockchains results in a production bottleneck (of blocks) which leads to high transaction fees that must be paid to miners. It was then in mid-2014, in Nxt and Bitcointalk forums, that the first ideas about distributed records based on DAGs began to be discussed, with the main objective of eliminating the fees and the production bottleneck present in Blockchains, creating a homogeneous environment where users verify network transactions.

In its initial form, there were still some questions that needed to be addressed by the new technology, such as: Where should the next block / transaction be added? And how / when will a transaction be considered confirmed (network consensus idea)? It was after intense discussion and thinking that the project known as IOTA emerged with a new technology called Tangle capable of resolving the issues raised. In this system, each transaction (represented by a graph node) must approve two previous transactions. Consensus can be provided in a distributed way, as suggested in the original technology article, using Markov Chain Monte Carlo (MCMC) algorithms.
